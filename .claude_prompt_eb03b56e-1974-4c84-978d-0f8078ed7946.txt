Phase 5: Graph Canvas Components

**Description**:
Implement GraphCanvas, GraphNode, GraphEdge, GraphInspector, TopologyNode, and the bezierPath/rectEdgePoint utility functions. Replace all mocks in GraphShowcase.tsx with real @heimdall/ui exports. This is the most complex phase and includes validation of the split-pane composition.

## Requirements
- FR-3.1/3.2: GraphCanvas — pannable (mouse-drag on canvas background) and zoomable (scroll/wheel, centered on cursor) SVG/HTML container; accepts nodes and edges arrays; manages pan/zoom state; exposes selectedNodeId and onNodeSelect as a controlled pair
- FR-3.3: GraphNode — positioned, selectable node rendered within GraphCanvas; accepts id, x/y, label, kind, domainColor token, and selected state; selected=true renders a 2px cyan border/ring per the active-state convention
- FR-3.4: GraphEdge — bezier-curve edge with a directed arrowhead at the target end; resolves node positions from GraphCanvas context; accepts an optional mid-path label; path updates when connected node positions change
- FR-3.5: GraphInspector — side-panel rendering key-value metadata and relationship links for the currently selected node; renders a configured empty state when no node is selected
- FR-3.6: Export bezierPath and rectEdgePoint utility functions as named exports from @heimdall/ui, consistent with the implementations in the reference applications
- FR-3.7: TopologyNode — a node variant styled as a bot/service card (title, role label, status dot, metric rows) for topology-layout graphs; uses MetricRow from Phase 1 or an equivalent inline metric row pattern
- GraphShowcase.tsx must validate split-pane composition: GraphCanvas + GraphInspector rendered inside the existing SplitPane component

## Design Guidance
