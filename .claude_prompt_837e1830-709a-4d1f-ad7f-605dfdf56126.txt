**Title**:
Phase 5: Feedback + Layout — Modal, Toast, CommandPalette, Panel, Drawer, SplitPane

**Description**:
Build the overlay, notification, and structural layout components. These components handle the most complex interaction patterns (focus trapping, keyboard navigation, portal rendering) and compose multiple primitives.

## Requirements
- `Modal`: portal rendering, focus trap on open, ESC dismissal, controlled `open`/`onClose`; 8px radius body, 10–12px radius modal container; shadow `0 24px 64px -16px rgba(0,0,0,0.55)`; 80–180ms ease animation
- `ConfirmDialog`: extends Modal; consequence-stating body text (e.g., 'Delete `cls_organism`? 47 individuals will be unlinked.'); danger action button variant
- `Toast`: success/error/warning semantic variants; auto-dismiss timer; stacking
- `CommandPalette`: fuzzy filter input, keyboard-navigable list, focus trap
- `Panel`: content section with optional header, border-only (no shadow)
- `Drawer`: 380px right drawer; slide-in animation 80–180ms ease
- `SplitPane`: two-pane (main + 380px right) and three-pane (220px left + flex middle + 320px right) variants
- `StatGrid`: 4-column grid, 14px gap, used to compose `StatTile`
- Visual regression tests against toast and modal reference preview cards

## Design Guidance
Implements the 'Phase 5: Feedback + Layout Components' section and the layout primitives table from the architecture design. ADR-004 governs controlled/uncontrolled patterns for Modal and Drawer.

## Acceptance Criteria
- [ ] Modal traps focus and dismisses on ESC; backdrop click behavior is configurable
- [ ] `ConfirmDialog` body text uses consequence framing (not 'Are you sure?')
- [ ] Toast auto-dismisses and stacks correctly for multiple simultaneous toasts
- [ ] `CommandPalette` filters list on input and navigates with arrow keys + Enter
- [ ] `SplitPane` renders correct column widths for both two-pane and three-pane variants
- [ ] Playwright visual snapshots match toast and modal reference preview cards within configured `maxDiffPixelRatio`
- [ ] No animation runs during Playwright screenshot capture (frozen via injected CSS)
- [ ] TypeScript compiles without errors
