**Title**:
Phase 2: PipelineCard rewrite

**Description**:
Replace the existing `PipelineCard` flat-prop API with a structured `pipeline` object, restore the flow strip visual that was lost during earlier integration, and update all consumers. This is the only breaking API change in v0.3.0.

## Requirements
- `PipelineCard` accepts a `pipeline` object with fields: `id`, `name`, `description`, `status` (`'running' | 'success' | 'idle' | 'failed'`), `target`, `flow: FlowNode[]`, `recent: { ingested, created, updated, errors }`, `tags: string[]`, `lastRun: string`
- Also accepts `onRun`, `onCancel`, and `compact` props
- Head region: `name` and `id` in mono, `description`, status chip, target chip, tags, Run/Cancel button, kebab menu
- Flow strip: colored node tiles connected by right-angle arrow lines; each `FlowNode` shows 22px icon tile, name, sub-label
- Foot region: 5-column grid with vertical dividers; `lastRun`, `ingested`, `created`, `updated`, `errors`; labels mono caps; values mono 13px tabular numerals; errors column rose when non-zero
- Any new flow-strip icons added to `ICONS` map in `Icon.tsx` (no inline SVG)
- Old flat props (`title`, `stages`, `statusLabel`, `statusColor`, `stats`) removed
- All test pages and docs references updated to new API
