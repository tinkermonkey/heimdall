**Title**:
Phase 3: Form components — Inputs, Select, Field, TriState

**Description**:
Build form primitives with consistent state handling and validation display. All form components must handle controlled and uncontrolled usage patterns and never embed application logic.

## Requirements
- `TextInput`: default, focus, error, disabled, and mono (monospace font) states
- `TextArea`: same states as TextInput
- `Select`: default, focus, error, disabled states
- `NumberInput`: same states as TextInput
- `Field` wrapper: composes Label + Input + optional required indicator + optional error message; label is sentence-case prose, never uppercase
- `TriState` checkbox: unchecked, checked, indeterminate states
- All inputs are controlled via `value`/`onChange` or uncontrolled via `defaultValue`
- Focus ring applied to all interactive elements via the system-wide focus ring spec
- Mono eyebrow label pattern available as a prop or via `Field` composition
- Visual regression tests against `components-inputs.html` reference preview card

## Design Guidance
Implements the 'Phase 3: Form Components' section of the architecture design. ADR-004 (Component State Ownership — controlled vs. uncontrolled) applies to all form components.

## Acceptance Criteria
- [ ] Each input type renders all documented states (default, focus, error, disabled)
- [ ] `Field` wrapper renders label, input, and error message in correct layout
- [ ] `TriState` checkbox renders all three states including indeterminate
- [ ] No form component makes API calls or imports from application state
- [ ] Playwright visual snapshot matches `components-inputs.html` reference within configured `maxDiffPixelRatio`
- [ ] TypeScript compiles without errors
