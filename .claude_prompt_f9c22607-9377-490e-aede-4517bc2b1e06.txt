**Title**:
[PR Feedback] Chart Components

**Description**:
Based on consolidated PR review, address the following in **Chart Components**:

- **BarChart division-by-zero and NaN propagation**: When all series have empty `data` arrays, `dataPointCount = 0` causes `barWidth = chartWidth / 0 = Infinity`. NaN/Infinity values propagate into SVG `rect` attributes, rendering an invisible chart. A single data point (line 149) produces a bar wider than the chart area. Equal-value datasets (`yRange = 0`) produce zero-height bars in BarChart and repeated y-axis labels in LineChart. `BarChart.tsx:80-81`, `BarChart.tsx:149`, `LineChart.tsx:59-61`
- **Sparkline stack-overflow-prone Math.min/max spread**: `Math.min(...points)` / `Math.max(...points)` throws `RangeError` on ~100K+ elements. Sister components `LineChart.tsx:43-54` and `BarChart.tsx:42-53` were already refactored to use loops with an explicit comment — Sparkline was missed. `Sparkline.tsx:31-32`
- **PieChart NaN from string coercion**: String values pass the `> 0` filter due to JS type coercion but cause string concatenation in the total calculation, producing NaN arc angles and broken rendering. `PieChart.tsx:35`
- **BarChart and PieChart have zero test coverage**: Both are rendered in `ChartsTestPage.tsx` fixtures but `charts-showcase.spec.ts` contains no tests, assertions, or visual snapshots for either. Non-trivial SVG rendering logic (arc math, grouped bar layout, empty-data guards) goes completely unregressed.
